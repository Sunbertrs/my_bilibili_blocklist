# my_bilibili_blocklist

自用Bilibili弹幕屏蔽规则

## 介绍

该弹幕屏蔽规则基于目前其他人已有的屏蔽规则，再主要以自身对弹幕的敏感/讨厌程度为基础进行编写，带有一些个人偏好，不一定保证适合所有人使用，但是欢迎提issue建议。

该项目有受到[这个项目](https://github.com/Griklit/Bilibili-Block-Word-Regex)的启发，运用了类似的项目结构。

这个项目里有一个简单的`blocklist.csv`，第一列是屏蔽规则，第二列是屏蔽类型（0为关键词屏蔽，1为正则表达式屏蔽，2为ID屏蔽），第三列是屏蔽理由/备注。

`blocklist2xml.py`脚本会将`blocklist.csv`中的内容转换成支持导入到B站的XML格式。

`json2csv.py`脚本用于将屏蔽规则导出的json给转换为csv，用于进一步对比操作。

`exist_check.py`脚本用于检查屏蔽规则是否已经存在于`blocklist.csv`中。

## 屏蔽偏好

- 政治敏感/内涵/外号

- 音游烂梗

- 手机厂商相关

- 孝子言论

- homo烂梗

- 无意义的发言

- 总之是你点了举报后，最终只会收到“您好，我们收到您举报的弹幕，我们将持续重点关注该账号”这种，人家大概率不理的规则。

总体上是往这些方向去屏蔽，因为这一类的弹幕最容易泛滥。遇到那种引战的，且确信举报后是会被处理的那就自己点一下了。

## 贡献规范

如果你想贡献屏蔽规则，请提交issue和pull request，说明你想添加的规则和理由。

commit messgae请使用conventional commit规范，一般更新规则请用`feat`，修改用`fix`，整合关键词用`refactor`。

一般情况下，只需修改`blocklist.csv`，**并且优先使用正则表达式屏蔽，将一小类有关联的关键词串起来。**

关键词串联最好不要大于10个，如果有，则拆分成多条栏目添加。

需要用到正则表达符的情况下请单起一条栏目。

如果你需要比较自己的屏蔽规则，可以通过视频页面的屏蔽设定→在屏蔽规则的表格内右键→导出json文件，然后使用`json2csv.py`脚本转换成csv格式，之后用`exist_check.py <刚才导出的csv文件>`来检查屏蔽规则是否已经存在。最终脚本给出的结果仅供参考。

## 构建方法

如需自行构建，运行`blocklist2xml.py`脚本即可。
